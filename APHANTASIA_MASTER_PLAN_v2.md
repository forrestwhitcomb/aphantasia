## The Product in One Sentence

Aphantasia is a whiteboard that turns simple shapes into beautiful, deployable websites — instantly, without prompts, without design skills, without waiting.

---

## Why This Will Win

The market as of March 2026:

- **Lovable** builds full apps from prompts. Beautiful output. But prompt-only — no spatial thinking, no visual control, slow iteration (full re-generation per change). Every Lovable site has "the Lovable look."
- **Google Stitch** (relaunched March 18, 2026) has an infinite canvas and voice input. But no deployment pipeline, no design system management, no micro-interactions, HTML/CSS only, and output quality is inconsistent.
- **v0** generates clean React components. But frontend-only, no canvas, no spatial input.
- **Figma** is precise but requires expertise and doesn't ship code.

**Aphantasia's thesis:** The fastest path from idea to deployed site is not typing a prompt — it's sketching. Draw a big rectangle at the top, Aphantasia renders a hero. Draw three boxes inside a bigger box, Aphantasia renders a feature grid. No prompt engineering. No learning curve. And the output is unique to your product because the design direction was generated from your context, not picked from a menu.

**Three structural advantages:**

1. **Instant visual feedback with no AI latency.** Layer 1 renders as you draw. The AI improves the output; it doesn't create it from scratch.
2. **Guaranteed quality floor.** The component library is beautiful at zero configuration. AI makes it better, never worse. Lovable and Stitch can produce ugly output on a bad generation. Aphantasia can't — the floor is high.
3. **Every project looks different.** Not because of theme presets (that's a college project). Because every project gets a generated Design DNA — a unique color palette, font pairing, spacing rhythm, decorative personality — produced from context before a single shape is drawn.

## The Design DNA System (This Is the Core Innovation)

### The Problem with Presets

The previous plan had 3 theme presets: Midnight, Editorial, Vivid. This means every dark SaaS site looks the same. Every portfolio looks the same. Users would quickly recognize "the Aphantasia look" — the exact same problem Lovable has.

Award-winning sites on godly.website and Awwwards don't look alike because each one has a bespoke design direction: a unique palette, unique typography choices, unique decorative elements, unique animation personality. That's what we need to generate.

### What Design DNA Is

Every Aphantasia project gets a `DesignDNA` object — a comprehensive, unique set of design tokens generated by AI from the user's context. This is NOT a preset. It's a bespoke direction computed per project.

```typescript
interface DesignDNA {
  // Generated color palette (NOT from a preset list)
  palette: {
    background: string        // e.g., "#0B1121" (deep navy)
    foreground: string        // e.g., "#E8E4DD" (warm off-white)
    accent: string            // e.g., "#06B6D4" (electric cyan)
    accentForeground: string  // e.g., "#FFFFFF"
    muted: string             // e.g., "#1E293B" (blue-gray)
    mutedForeground: string   // e.g., "#94A3B8"
    card: string              // e.g., "#0F1A2E"
    cardForeground: string    // e.g., "#CBD5E1"
    border: string            // e.g., "rgba(255,255,255,0.08)"
  }

  // Selected from a curated library of 20+ pairings
  typography: {
    headingFamily: string     // e.g., "Clash Display"
    bodyFamily: string        // e.g., "Satoshi"
    headingWeight: number     // e.g., 700
    headingLetterSpacing: string // e.g., "-0.04em"
    scale: 'dramatic' | 'balanced' | 'compact'
    // dramatic: hero h1 clamp(56px, 10vw, 120px)
    // balanced: hero h1 clamp(40px, 7vw, 80px)
    // compact: hero h1 clamp(32px, 5vw, 56px)
  }

  // Spatial personality
  spacing: {
    density: 'spacious' | 'balanced' | 'tight'
    sectionPadding: string    // e.g., "clamp(80px, 12vw, 140px)"
    contentMaxWidth: string   // e.g., "1100px"
    cardPadding: string       // e.g., "32px"
    gridGap: string           // e.g., "24px"
  }

  // Surface treatment preferences
  surfaces: {
    hero: 'flat' | 'gradient-mesh' | 'grain' | 'accent-wash' | 'radial-glow'
    sections: 'alternating' | 'uniform' | 'contrasting'
    cards: 'elevated' | 'bordered' | 'glass' | 'flat' | 'accent-top'
  }

  // Decorative personality — what makes it feel unique
  decorative: {
    style: 'geometric' | 'organic' | 'minimal' | 'editorial' | 'grid-overlay' | 'gradient-blobs'
    // geometric: circles, lines, angular shapes in backgrounds
    // organic: blobs, waves, fluid shapes
    // minimal: nothing — whitespace is the decoration
    // editorial: rules, dividers, typographic ornaments
    // grid-overlay: subtle dot or line grid behind sections
    // gradient-blobs: large soft gradient circles (like Stripe)
    intensity: 'subtle' | 'moderate' | 'bold'
  }

  // Animation personality
  motion: {
    level: 'none' | 'subtle' | 'expressive'
    entrance: 'fade-up' | 'fade-in' | 'slide-in' | 'scale-up' | 'none'
    hover: 'lift' | 'glow' | 'scale' | 'border-accent' | 'none'
    staggerDelay: string      // e.g., "0.08s"
    duration: string          // e.g., "0.6s"
  }

  // Button personality
  buttons: {
    radius: string            // "0px" (sharp) to "9999px" (pill)
    style: 'solid' | 'outline' | 'ghost' | 'gradient'
    size: 'compact' | 'standard' | 'large'
  }

  // Image treatment
  images: {
    radius: string            // card corner radius for images
    treatment: 'natural' | 'duotone' | 'grayscale' | 'saturated'
    frame: 'none' | 'browser' | 'phone' | 'shadow'
  }

  // The mood document slug for Deep Render (AI prompt augmentation)
  moodSlug: string           // e.g., "dark-saas", "editorial", "bold-brand"
}
```

### How Design DNA Gets Generated

**Step 1: Context input (the FIRST interaction)**

The context widget is not a side panel. It's the **first thing the user interacts with** when they open Aphantasia. Like Lovable's "what do you want to build?" prompt — but instead of generating code, it generates a design direction.

The user types something like:
- "A privacy-focused email client for journalists"
- "My wedding photographer portfolio"
- "Landing page for a fintech startup targeting Gen Z"
- "Restaurant website for a high-end Japanese restaurant in Stockholm"

**Step 2: AI generates Design DNA (~2-3 seconds)**

The extraction prompt takes the user's freeform text and produces both `StructuredContext` (product information) and `DesignDNA` (visual direction) in a single call.

The prompt is highly specific:
- "Generate a unique color palette. NEVER use the same palette twice. Pull inspiration from the product's industry, audience, and mood. A Japanese restaurant gets deep blacks with warm amber accents. A Gen Z fintech gets electric gradients with neon coral. A journalist email tool gets muted navy with subtle teal."
- "Select a font pairing from this curated library: [list of 20+ pairings]. Pick the one that best matches the product's personality. Developer tools get geometric sans. Luxury brands get serif headings. Personal sites get friendly rounded type."
- "Choose decorative elements. A SaaS dashboard gets geometric shapes — angular, precise. An organic skincare brand gets flowing blobs. A law firm gets nothing — whitespace is the decoration."

**Step 3: Design preview card appears**

Before the user touches the canvas, a preview card renders showing:
- The generated color palette (swatches)
- The font pairing (sample heading + body text)
- A mini-preview of how a hero section would look with this DNA
- "Use this direction" / "Regenerate" / "Customize" buttons

**Step 4: DNA flows into everything**

Once confirmed, the Design DNA converts to CSS custom properties and applies to the entire render pipeline:

```css
:root {
  --aph-bg: #0B1121;
  --aph-fg: #E8E4DD;
  --aph-accent: #06B6D4;
  --aph-muted: #1E293B;
  --aph-card: #0F1A2E;
  --aph-border: rgba(255,255,255,0.08);
  --aph-heading-font: "Clash Display", sans-serif;
  --aph-body-font: "Satoshi", sans-serif;
  --aph-section-padding: clamp(80px, 12vw, 140px);
  --aph-card-radius: 12px;
  --aph-button-radius: 8px;
  /* ... 30+ more tokens */
}
```

Every component reads these tokens. Every Layer 1 render uses them. Every Layer 2 fill respects them. The DNA is the design system for THIS project, not a global preset.

### The Decorative System (What Makes Each Site Feel Unique)

Beyond palette and typography, sites feel unique because of decorative elements. The `decorative` field in DesignDNA drives a set of CSS/SVG background treatments that are applied per-section:

**Geometric:** Subtle circles, lines, or angular shapes positioned as background decorations. Think Linear's homepage — the grid lines, the subtle glow effects.

**Organic:** Soft gradient blobs, wavy dividers between sections, fluid background shapes. Think Stripe's homepage — the huge gradient mesh behind the hero.

**Minimal:** Nothing. Whitespace, typography contrast, and color are the only tools. Think Notion's marketing site — almost no decoration, just perfect spacing.

**Editorial:** Horizontal rules, typographic dividers (ornamental lines between sections), pull-quote styling, drop caps. Think New York Times or Monocle.

**Grid Overlay:** A subtle dot or line grid behind certain sections, creating a technical/structured feeling. Think Vercel's dashboard aesthetic.

**Gradient Blobs:** Large, soft, multi-color gradient circles or shapes positioned behind sections. High-impact, modern, very 2025-2026.

Each section component checks `DesignDNA.decorative.style` and renders the appropriate background treatment. This is CSS — no images, no external assets, instant rendering.

### The Deep Render Unlock

When a user hits Deep Render, the AI receives the full Design DNA as structured input — not vague instructions. The Deep Render prompt says:

"This project uses the following design direction: [full DesignDNA JSON]. Create a bespoke implementation that pushes this direction further. Add GSAP scroll animations. Add custom SVG decorations that match the decorative style. Add micro-interactions on every interactive element. Make it award-worthy. But stay within the DNA — the palette, the typography, the spatial personality. The DNA is the guardrails; your creativity is everything else."

This means Deep Render output is unique per project AND visually coherent with the Layer 1/2 output the user has been working with. It's an enhancement, not a replacement.

## Progressive DNA: Context Accumulates, Design Sharpens

### The Core Principle

**The canvas works beautifully from the very first shape with zero context.** There is no gate, no onboarding form, no "describe your product before you can draw." The user opens Aphantasia, sees a blank canvas, draws a rectangle, and instantly sees a beautiful rendered section.

Context is gathered progressively from multiple sources — the canvas itself, the context widget, reference screenshots, notes — and each signal sharpens the Design DNA. The DNA is never empty; it starts with an elegant default and evolves as the user works.

### The Default DNA (Zero Context)

When no context has been provided, Aphantasia uses a carefully designed default DNA that looks professional, clean, and elegant — NOT generic wireframe-gray. Think of it as "the best possible site before we know anything about you."

The default DNA:
- Dark background (#0C0C0C), warm off-white foreground (#EBEBEB), subtle teal accent
- Clean geometric sans heading font (Satoshi or equivalent), humanist body font
- Spacious layout with generous section padding
- Minimal decorative style — whitespace as the primary design element
- Subtle entrance animations (fade-up on scroll)
- Slightly rounded buttons (8px radius), standard size

This default must look good enough that a user could deploy it as-is. It's the "little black dress" of design directions — always appropriate, never embarrassing.

### Phase 1: Canvas Signals (Passive, Automatic)

As the user draws and labels shapes, the canvas itself becomes a context source. Aphantasia watches for signals and refines the DNA in real time:

**Labels as signals:**
- User labels a shape "Pricing" → DNA infers this is likely SaaS, adjusts feature grid defaults
- User labels a shape "Menu" → could be navigation or restaurant menu — spatial position disambiguates
- User types "FocusFlow" in a text element → DNA captures a product name, placeholder copy starts using it

**Notes as signals:**
- User adds a note outside the frame: "dark mode" → DNA shifts to dark palette if not already
- Note says "should feel like Linear" → DNA shifts toward geometric, dark, minimal
- Note says "playful, bright, for kids" → DNA shifts to warm colors, rounded shapes, friendly type

**Structural signals:**
- User creates many card shapes → DNA infers grid-heavy layout, tightens grid gap
- User creates a single large hero + a few sections → DNA infers editorial/landing page, increases section padding
- User adds an image shape → DNA adjusts image treatment tokens

These passive signals use lightweight rules (no AI call needed for most) and update the CSS custom properties immediately. The preview re-renders in <50ms. The user sees the design evolve as they build — shapes are rendered with DNA that increasingly reflects their intent.

### Phase 2: Context Widget (Active, Optional, Any Time)

The context widget lives in a collapsible panel on the canvas — always accessible, never forced. A user can open it at any point:

- Before drawing anything (Lovable-style: "I'm building a privacy-focused email client for journalists")
- After sketching a rough layout (the wireframe exists, now add meaning)
- After seeing the Layer 1 render and wanting it to feel more specific

When the user writes in the context widget:

1. **AI generates a full DesignDNA** from the description + any signals already gathered from the canvas (~2-3 seconds)
2. A **design direction preview** appears showing palette swatches, font samples, and a mini component preview
3. User can "Apply," "Try another" (regenerate with different creative choices), or "Customize" (manual token adjustment)
4. On Apply, the DNA replaces the current DNA — the preview transforms. Same shapes, dramatically different look.

**The context widget also gathers context from the canvas.** When it runs the extraction prompt, it includes everything the user typed in the widget, all shape labels, all notes on and off frame, the semantic tags of existing shapes, and the output type. This means the extraction gets smarter as the canvas gets richer.

### Phase 3: Reference Screenshots (Active, Optional, Any Time)

Users can drop screenshots into the reference widget at any time:

1. Vision API analyzes: dominant colors, typography style (serif/sans/mono), layout density, contrast level, component patterns
2. Extracted values **override specific DNA tokens** — accent color, heading font family, background lightness
3. Structural DNA (spacing, decorative style, animation) only changes if the reference clearly suggests it
4. Preview updates immediately (<2s)

**This is how you get "make it look like Linear" without typing those words.** Drop a Linear screenshot, Aphantasia extracts the dark bg, clean sans typography, and geometric decorative elements, and overwrites the DNA accordingly.

### The DNA Lifecycle

```
Open Aphantasia
  │
  ├─ Default DNA loads (elegant, neutral, professional)
  │
  ├─ User draws shapes → Layer 1 renders with default DNA
  │  └─ Canvas signals (labels, notes) refine DNA passively
  │
  ├─ User opens context widget (optional, any time) → types description
  │  └─ AI generates full DNA from context + canvas signals → user previews → applies
  │     └─ Preview transforms — same layout, new visual identity
  │
  ├─ User drops reference screenshot (optional, any time)
  │  └─ Extracted tokens override DNA → preview updates
  │
  ├─ User hits Render (Layer 2)
  │  └─ AI fills content + selects variants within current DNA
  │
  └─ User hits Deep Render ✦
     └─ AI creates bespoke HTML/CSS/JS guided by full DNA + mood document
```

**At every stage, the output is beautiful.** The default DNA ensures the floor is high. Each layer of context makes it more specific, more unique, more "theirs" — but at no point does the user see a blank preview or a wireframe waiting for context.

### DNA Persistence and Editing

The DNA is part of the saved project state (localStorage alongside CanvasDocument). When a user returns, they see exactly what they left.

A "Design Direction" pill in the toolbar shows the current DNA state:
- Click to expand: palette swatches, font names, decorative style
- "Customize" opens a DNA editor: manual color picker, font selector, spacing controls, decorative style toggle
- "Regenerate" re-runs the AI generation from current context + canvas signals
- "Reset" returns to default DNA

## Component Library (The Quality Floor)

### The Non-Negotiable Standard

Every section component must satisfy this test:

> Open Aphantasia. Type "SaaS productivity tool." Confirm the design direction. Draw 5 rectangles on the canvas in a typical landing page layout. Do NOT label them. Do NOT use the component browser. Do NOT trigger Layer 2 AI.
>
> **The result must look like a real website that someone paid a designer to build.** Not a wireframe. Not a template. A site you'd show to an investor.

This is the quality floor. Layer 2 and Deep Render make it better — but the floor must already be impressive.

### The 18 Sections (same as previous plan, with tighter spec)

Each section component:

1. Takes typed props + `DesignDNA` object
2. **Renders beautifully with ZERO props** using smart defaults and DNA tokens
3. Renders with partial props (Layer 1 fills what it can)
4. Renders with full props (Layer 2 enriched state)
5. Uses ONLY CSS custom properties derived from DNA — zero hardcoded colors
6. Implements ALL layout variants for its type
7. Includes the decorative background treatment matching `DNA.decorative.style`
8. Includes entrance animations matching `DNA.motion`
9. Includes hover interactions matching `DNA.motion.hover`
10. Is responsive with real layout changes (not just `max-width: 100%`)
11. Uses the DNA's font pairing (heading + body)
12. Uses the DNA's button personality (radius, style, size)

**Core sections:** Nav, Hero, Feature Grid, Text+Image Split, CTA, Footer
**Conversion sections:** Pricing, Testimonials, Logo Cloud, Stats, Newsletter
**Supporting sections:** FAQ, Team, Comparison, Portfolio, E-commerce, Event, Generic

(See previous plan for full variant lists and label triggers — those haven't changed.)

### What Makes a Component "Industry Standard"

Study these sites before building each component:

- **Hero:** Linear.app, Raycast.com, Vercel.com/home, Stripe.com
- **Feature Grid:** Notion.so, Figma.com, Arc.net
- **Pricing:** Vercel.com/pricing, Linear.app/pricing, Stripe.com/pricing
- **Testimonials:** Webflow.com, Framer.com, Monday.com
- **Nav:** Stripe.com, Linear.app (minimal + glassmorphism)
- **Footer:** Stripe.com (mega footer), Linear.app (clean columns)

The details that separate industry-standard from template:
- Typography hierarchy is dramatic (h1 and body text feel like different typefaces even if they're the same family — weight and scale contrast does this)
- Hover states have personality (not just color change — lift, glow, scale, border-accent)
- Cards have subtle depth (not just box-shadow — background contrast + border + hover elevation change)
- Buttons feel solid (proper padding, clear hierarchy between primary/secondary/ghost)
- Sections have rhythm (padding varies — hero gets more, dense grids get less)
- Real placeholder content (not "Lorem ipsum" but "Ship faster with AI-powered development" and "10,000+ teams trust FocusFlow")

## The Three Render Tiers (Unchanged)

**Layer 1 — Instant (0ms, no AI)**
Rules engine → semantic tags → DNA-informed default variants → render.
The DNA makes Layer 1 output look bespoke. This is where the magic is.

**Layer 2 — Variant + Content (~3-5s, streaming JSON)**
Claude receives canvas layout + StructuredContext + DesignDNA + mood document.
Returns JSON with coherenceStrategy + per-section variant selections + content.
Content is specific to the product ("Ship encrypted email in minutes" not "Get started today").

**Deep Render — Opt-in (~15-20s, bespoke HTML/CSS/JS)**
Claude generates complete custom HTML with GSAP, custom SVG decorations, and award-worthy output.
Uses Layer 2 props + DesignDNA as its brief.

## Design Mood Documents (6 for v1)

These are 200+ line markdown files that give Deep Render extremely specific aesthetic instructions. They are NOT presets for Layer 1 — they're creative briefs for the AI.

The mood is auto-selected from StructuredContext:

| Mood | Auto-Selected When |
|------|-------------------|
| **Dark SaaS** | contentType: saas + tone: confident + DNA has dark bg |
| **Editorial Elegance** | contentType: portfolio/editorial + DNA has serif heading |
| **Bold Brand** | contentType: saas + tone: playful/urgent + DNA has vibrant accent |
| **Clean Commerce** | contentType: ecommerce |
| **Warm Personal** | contentType: personal/nonprofit |
| **Agency Showcase** | contentType: agency |

But the mood document works WITH the DNA, not instead of it. The mood says "use dramatic typography scale and scroll-triggered reveals" while the DNA specifies WHICH fonts and WHICH colors. They're complementary layers.

## Claude Code Session Guide

### Session Philosophy

Every session ships one complete, tested piece. No session should end with "this will be wired up later." If you're building a component, it renders in the preview. If you're building an API, it returns real data. If you're building UI, it's interactive.

### Critical Session: Default DNA + Generation Prompt (Days 1-2)

These are the two most important pieces of the product.

**The default DNA** determines whether zero-context output looks professional or like a wireframe. Spend serious time on this. The default palette, the default font pairing, the default spacing rhythm — these are what every user sees first. They must be beautiful and opinionated without being alienating. Test by drawing 5 shapes with the default DNA and asking: "Would I show this to a client?"

**The DNA generation prompt** determines whether contextual output looks unique or samey. It must NEVER produce similar palettes for different products. ## Architecture Decisions (All Locked)

Everything from CONTEXT.md remains in force, plus:

1. **Design DNA is generated per-project, NEVER selected from presets.** There are no theme presets. There is one font library, one color generation system, one decorative system — and the AI selects from the continuous space, not from buckets.
2. **Context widget is available from the start, never forced.** It's a collapsible panel, not a gate. Users can draw immediately with zero context. The DNA starts beautiful and sharpens as context accumulates from the canvas itself, the widget, and reference screenshots.
3. **The component library is the competitive moat.** Every component must be independently beautiful. AI improves; it doesn't save.
4. **Design DNA converts to CSS custom properties.** Components NEVER read DNA fields directly — they read CSS vars. This means DNA changes instantly update the entire render.
5. **Freeform sketching is primary.** Component browser is secondary. Every component must be reachable via spatial inference + labels.
6. **Layer 1 with DNA must look professional.** If Layer 1 output isn't impressive, the DNA generation or the component library needs to be better. Don't paper over it with AI.

## Success Criteria (Ruthless)

v1 ships when ALL of these are true:

1. **Generate DNA for "SaaS productivity tool" and "wedding photographer portfolio" → the two design directions look NOTHING alike.** Different palette, different fonts, different decorative style, different spatial rhythm. Not "same template, different colors."

2. **Open Aphantasia with zero context, sketch 5 shapes → the result looks like a professionally designed website.** No context widget, no labels, no component browser. Just shapes and the default DNA. The default must be that good.

3. **Now open the context widget and type "privacy-focused email for journalists" → the preview transforms.** Same shapes, completely different visual identity. The palette, fonts, decorative elements, and placeholder copy all shift to match the product.

4. **Same canvas, different context → visibly different site.** Not just different words — different typography, different colors, different decorative elements, different animation style.

5. **Drop a Linear.app screenshot as reference → the output shifts to match.** Dark mode, clean geometric style, specific accent color.

6. **Layer 2 generates copy that sounds like it was written for THIS product.** Not generic startup copy. "Ship encrypted email in minutes" for the journalist tool, not "Get started with our product today."

7. **Deep Render produces output that would get noticed on godly.website.** Award-worthy. Custom animations. Bespoke decorative elements. This is the premium tier and it must be premium.

8. **Full deploy loop works: context → DNA → sketch → render → GitHub → Vercel.** End to end, no manual steps.

9. **Canvas state survives refresh.** localStorage persistence.

10. **The product feels fast.** Layer 1 renders under 50ms. Layer 2 under 5 seconds. DNA generation under 3 seconds. No loading spinners that last longer than a heartbeat.

If those ten things work, ship it. If any of them don't, fix them before shipping.
